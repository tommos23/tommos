Software Engineering Crossover Project

"This module introduces the Software Engineering concepts that are needed to develop software systems that can meet basic functional requirements within a well-defined problem domain. It covers the main steps in the process of developing such systems: requirements analysis, system specification, system design, system implementation and system testing. The module takes a model-based approach, in that at each stage it explains the various models of the software that need to be created, the notations to be used for the models (and particularly selected parts of the Unified Modelling Language, or UML), and the basic tools and techniques for producing these models. A major part of the module involves students working on a team project (the crossover project) to develop examples of the software systems being discussed, so giving practical experience in teamwork and in managing such projects." From the Software Engineering Crossover Project Website

In this project I have developed a simple program in a team that accepts input from the user and performs calculations on them then prints the result.

This is the code used for one of the applications it was written in java using a text editor. The documentation for the design and testing was written in Microsoft word. The design was done by the whole team using pseudo code and the testing was done by inputting sets of values to check that the program worked correctly for all of the constraints.


This is what the program looks like when it runs

The user enters a value between 1 and 9999 and then the program calculates which digit in the number has the highest value.

CrossOver project and assessment
The course marks are broken down into the following categories:

  • Assignment 1 (simple programming group task) 10%
  • Assignment 2 (requirements document) 20%
  • Assignment 3 (design document) 20%
  • Assignment 4 (implementation and unit testing) 30%
  • Assignment 5 (presentation + system/acceptance testing) 10+10%
Processes involved in these activities (minutes, agendas, interactions) meant to be 10-20% of each of the above marks
